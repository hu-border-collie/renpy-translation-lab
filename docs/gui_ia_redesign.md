文档地图：[docs/README.md](README.md) · 现状说明：[GUI 工作台](gui_workbench.md)

> **状态**：计划已修订；GitHub Epic/子 issue 已创建；**P0a (#158) 已合并**；**P0b (#159) 实现中**。  
> **Epic**：[#157](https://github.com/hu-border-collie/renpy-translation-lab/issues/157)

## 1. 背景与问题

### 1.1 现状结构

顶层三个 Tab：**工作台** · **设置** · **诊断日志**。

工作台采用 **「一套壳 × 八种模式」**：

- 两级下拉（任务类型 × 子任务）切换 `WorkMode`
- 同一主按钮变形为：开始翻译 / 提取关键词 / 预建库 / 生成模板…
- 同一组内层 Tab（环境检查 / 进度 / 写回）标签与按钮随模式变形
- 任务运行时强制切到诊断页并放大日志

### 1.2 主要摩擦（盘点结论）

| # | 问题 | 影响 |
|---|------|------|
| 1 | 主按钮语义过多 | 用户不确定当前会执行什么 |
| 2 | 写回 Tab 永远存在 | 同步/预建只有说明，却占「结果」位 |
| 3 | 能力散落三处 | 拆分在诊断、状态表在进度、提交剩余包在按钮行；glossary 双入口 |
| 4 | 运行时强切诊断 | 主路径被打断；进度与日志割裂 |
| 5 | 项目入口双轨 | 工作台选目录 vs 设置→工作区；文档写「唯一入口」但代码另有入口 |
| 6 | 上下文配置撕裂 | 批量 RAG 在「上下文」，同步 RAG / 剧情记忆在「高级」 |
| 7 | 准备 / 翻译 / 分析混装 | 环境检查与关键词、预建同级并列 |
| 8 | 恢复工具堆在写回 | 补译/修补/补救挤成按钮墙，淹没「写回翻译」 |

### 1.3 能力全景（重组对象）

```text
项目生命周期     选项目 · 准备工作目录 · 环境检查 · 生成模板
主翻译           批量翻译（check→apply）· 同步翻译（可能直接写文件）
批量侧工具       拆分 · 试跑 probe · A/B · 补译 · 修补
术语             关键词提取 · 合并到 glossary
订正             订正预览 · 写回订正（独立闸门）
上下文库         开关（设置）· 预建 RAG / 原文索引
配置             密钥 · 模型 · 项目 · 上下文 · 高级
诊断             任务上下文 · 命令参考 · 任务记录 · 原始日志
```

这些能力**不是**同一维的 8 个「子任务」，却被塞进同一套下拉。

## 2. 目标与非目标

### 2.1 目标

1. **按用户目标分区**，不再用「模式枚举」驱动整页变形。
2. **主路径极简**：选项目 → 检查 → 批量翻译 → 可写回 → 写回。
3. **次要任务有固定入口**，不抢主按钮。
4. **同一能力原则上只出现一次主入口**（允许次要快捷，须标明从属关系）。
5. **运行时默认不抢焦点**：日志在工作台底部抽屉；诊断页改为主动打开。
6. **结果区随任务类型换面板**，不再永远叫「写回」。
7. **设置分层**：开关类上下文配置归位；高级只留调参。

### 2.2 非目标

- 不改 CLI 子命令、参数语义、manifest schema。
- 不改 `safe` / `warn` / `block` 写回闸门规则。
- 不引入新的翻译能力或新的 work mode 语义（只重排入口）。
- 不在本 epic 内做安装器 / 多租户 per-game 配置拆分。
- 不强制重写 `app.py` 为多文件（允许分阶段抽取，见 §8）。

### 2.3 已拍板决策

| 议题 | 决定 |
|------|------|
| 工作台导航形态 | **左侧固定任务列表**（非两级下拉，非顶部分段控件） |
| 日志位置 | **工作台底部可折叠抽屉**；任务运行时**默认不**自动切到诊断 Tab |
| 交付方式 | 计划书（本文 SSOT）→ 后续拆 issues → 按 **P0a → P0b → P1 → P2 → P3** 实现 |

## 3. 目标信息架构

### 3.1 应用壳

```text
┌──────────────────────────────────────────────────────────────────┐
│ Ren'Py Translation Lab · 图形工作台                              │
│ 项目：[work 路径 ··············] [切换项目 ▾]   API 状态 · 状态条 │
├────────────┬─────────────┬───────────────────────────────────────┤
│   工作台    │    设置     │           诊断与工具                   │
└────────────┴─────────────┴───────────────────────────────────────┘
```

**全局项目栏**（所有顶层 Tab 可见）：

- 只读路径 + **切换项目**（打开工作区选择器 / 跳转设置→工作区，或轻量弹层）
- 可选：API Key 状态芯片（只读，点进设置→密钥）
- 工作台内不再把「选目录」当作主叙事；文件夹自由选择可降级为「高级：指定本地目录…」或保留在切换流程中

全局栏**不是纯布局**：当前项目选择只在工作台内（`select_btn`），运行中禁用绑在 `select_btn`、settings workspace 跳转与 registry panel 上（`_set_task_running`）。上提到全局后，必须统一状态契约，见 **§3.1.1**。P0 将其单独放在 **P0b**，不得与日志焦点拆分混在同一模糊验收里。

**顶层 Tab 改名**：

| 现名 | 目标名 | 说明 |
|------|--------|------|
| 工作台 | 工作台 | 保留 |
| 设置 | 设置 | 保留 |
| 诊断日志 | **诊断与工具** | 强调只读上下文 + 实验工具 + 完整日志 |

#### 3.1.1 全局项目栏状态契约（P0b 必做）

今日代码事实：

| 控件 / 入口 | 位置 | 运行中 | 切换语义 |
|-------------|------|--------|----------|
| `select_btn`「选择游戏目录…」 | 仅工作台 `project_frame` | `_set_task_running` → `setEnabled(False)` | 立即 `set_game_root` |
| 设置 → 工作区 `GamesRegistryPanel` | 设置页 | panel `setEnabled(False)` | 立即 `set_game_root` + registry |
| `settings_go_workspace_btn` | 设置 → 项目 | 运行中禁用 | 仅跳转导航，不改 root |
| 设置脏状态 leave guard | 离开设置 Tab / 开始任务 / registry 切换 | — | 保存 / 丢弃 / 取消 |

**全局栏实现时必须写清并测通的关系：**

1. **运行中禁用**  
   - 全局栏的「切换项目」与任何「指定本地目录」在 `_task_running` 时禁用（与现 `select_btn` 同策略）。  
   - 禁用对象是**逻辑入口**（方法或按钮），不是只藏工作台内旧按钮却留全局栏可点。

2. **settings 脏状态**  
   - 从全局栏切换 `game_root` 时，若设置页有未保存改动，必须走与现 registry 切换相同的 **保存 / 丢弃 / 取消** 路径（不得静默丢配置或静默带着脏 UI 换 root）。  
   - 取消则不改 `game_root`、不改全局栏显示。

3. **registry 与自由选目录**  
   - **主路径**：切换项目 → 工作区列表（registry）选中并切换。  
   - **次路径**：「指定本地目录…」可保留，须即时 `set_game_root`（与今日一致）；不要求自动写入 registry，但路径展示与 doctor/工作台一致。  
   - 设置 → 工作区仍是总表维护与深度刷新的主界面；全局栏是**快捷切换**，不是第二套 registry 编辑器。

4. **`game_root` 同步**  
   - 唯一权威仍是 `translator_config.json` 的 `game_root` + `ProjectState`。  
   - 全局栏路径、工作台（若仍残留只读行）、设置 → 项目只读值必须同源刷新（现有 `_refresh_project_label` 一类路径扩展为全局栏）。  
   - 切换成功后：reload 设置 UI、清空/stale doctor、与现 `set_game_root` 后行为一致（含 asset path sync）。

5. **设置页工作区跳转**  
   - 「在工作区切换…」继续只负责 `settings_nav` 跳到工作区分区。  
   - 全局栏「切换项目」可直接打开同一 panel 的选择体验，或 `setCurrentWidget(设置) + 工作区 row`；**禁止**第三套切换状态机。

6. **工作台内旧 `project_frame`**  
   - P0b 目标：去掉重复主叙事（或降为只读镜像）。若暂留 `select_btn`，必须与全局栏共用同一 enable/disable 与同一 `_on_select_project` / switch 入口，避免双按钮不同门控。

**风险定性**：全局栏为 **中**（状态与 leave-guard），不是「低–中布局活」。故 **P0a 不包含全局栏**。

### 3.2 工作台：左导航 + 右内容 + 底抽屉

```text
工作台
├── 左侧「任务」导航（固定列表）
│   ├── ★ 批量翻译          ← 默认选中
│   ├── 同步翻译
│   ├── 关键词 / 术语
│   ├── 订正
│   └── 上下文库
│
├── 右侧内容区（随左侧切换整页，不共用变形主按钮）
│
└── 底部「运行日志」抽屉（可折叠）
    ├── 标题栏：运行日志 · 展开/折叠 · 清空 · 在诊断中打开
    └── QTextEdit（与现 log_view 同源或镜像）
```

#### 3.2.1 批量翻译（默认主路径）

用**阶段**代替平级的「环境检查 / 进度 / 写回」三 Tab：

```text
[ 1 准备 ] ── [ 2 执行 ] ── [ 3 结果 ]
```

| 阶段 | 内容 | 主行动 |
|------|------|--------|
| **准备** | doctor 摘要 + 推荐下一步；准备工作目录；缺模板时「生成翻译模板」 | 环境检查 / doctor 推荐行动 |
| **执行** | 时间线 + 进度 + 事实；开始 / 继续 / 查询状态 / 停止；拆分包状态表（有则显示） | 开始翻译 · 继续 · 停止 |
| **结果** | 安全徽章；**主按钮：写回翻译**；折叠「问题处理」 | 写回翻译 |

**结果 · 问题处理**（折叠，默认收起；仅 `warn` / 失败等需要时自动展开或角标提示）：

- 查看问题清单  
- 生成 / 查看补译包 · 继续补译  
- 同步修补  
- 重新检查  
- 查看写回失败报告  
- 打开命令参考（原「补救命令」跳转）

**执行 · 高级工具**（次要按钮组或「高级 ▾」菜单，P2 归位）：

- 拆分翻译包  
- 试跑样本请求  
- 提交剩余包（仅拆分场景）  

**翻译 A/B 对比**：偏实验 → 留在「诊断与工具」；可选在执行阶段「实验」折叠中放快捷入口（P2/P3）。

#### 3.2.2 同步翻译

```text
警告条：可能直接修改项目文件，请先备份
[ 开始同步翻译 ] [ 停止 ]
进度 + 结果说明（无「写回翻译」按钮）
```

不复用批量三阶段的写回闸门 UI。

#### 3.2.3 关键词 / 术语

```text
模式：批量 | 同步
[ 提取关键词 ] [ 继续（批量）] [ 停止 ]
结果：报告摘要与路径 · [ 合并到 glossary ]  ← 唯一主入口
```

#### 3.2.4 订正

```text
模式：批量 | 同步
[ 生成订正预览 ] [ 继续（批量）] [ 停止 ]
结果：预览摘要 · [ 写回订正 ]  ← 与「写回翻译」文案/样式区分
```

#### 3.2.5 上下文库

```text
状态卡：
  记忆库：开/关 · 记录数 · 路径 · [预建记忆库]
  原文索引：开/关 · 片段数 · 路径 · [预建原文索引]
链到设置 → 上下文（开关须保存后才能预建，规则不变）
```

预建**不再**与翻译/关键词并列成「子任务」。

### 3.3 设置导航重组

| 顺序 | 分区 | 内容调整 |
|------|------|----------|
| 1 | 工作区 | 总表扫描 / 切换（全局项目栏的深度入口）；即时写 registry |
| 2 | 项目 | glossary / tl / include / prepare；`game_root` 只读 + 链到工作区 |
| 3 | 密钥 | 即时写 `api_keys.json` |
| 4 | 模型 | 同步 + 批量模型、thinking |
| 5 | **上下文** | **批量与同步** RAG 开关、原文索引、**剧情记忆开关**、存储位置、与预建相关的主开关 |
| 6 | 外观 | 主题 |
| 7 | 高级 | 吞吐、chunk、检索参数、store 路径覆盖等纯调参 |

**设置语义小改进**（可与 P2 同批或紧随）：

- 在「工作区」时：底部「保存 / 恢复推荐值」禁用或隐藏，避免假保存（「重新加载」仅重载 `translator_config`，文案说清）
- 无 `game_root` 时：允许保存外观 / 模型（若实现成本高可标为 P3 可选）

### 3.4 诊断与工具（瘦身）

定位：**只读上下文 + 完整日志 + 实验工具**，不承载主流程操作。

```text
诊断与工具
├── 工具条
│   ├── 刷新上下文
│   ├── 翻译 A/B 对比
│   └── 清空日志
│   （拆分 / 试跑 / 合并 glossary 主入口迁出）
├── 上：任务上下文 | 命令参考 | 任务记录
└── 下：原始输出（与工作台抽屉同源缓冲）
```

**运行时行为**（细节与 API 见 **§5 P0a**、**§6.3**）：

- 从**工作台**发起的任务：默认留在工作台，调用 `_show_workbench_log_drawer()` 展开抽屉；**不**自动切换主 Tab 到诊断
- 从**诊断页**发起的 probe / split / A/B：调用 `_expand_diagnostics_log()` 放大诊断日志区（可保持在诊断 Tab）
- 用户可点「在诊断中打开」看上下文 / 命令参考 / 完整布局
- 任务结束后：不强制改抽屉状态；若曾为运行自动展开，可选记住用户手动折叠

## 4. 用户路径（验收用）

### 4.1 新手 · 批量主路径

```text
启动
  → 全局栏选择 / 切换项目
  → 设置：密钥 + 模型 → 保存
  → 工作台 · 批量翻译 · 准备：环境检查
  → （可选）准备工作目录 / 生成模板
  → 执行：开始翻译 →（排队）继续 / 查询状态
  → 结果：可写回 → 写回翻译
```

**验收**：全程不必打开诊断 Tab（P0a 后工作台任务不强制跳转；诊断内工具仍可在诊断页放大日志）；不必理解 8 种模式；默认看不到补译按钮墙（P0b 折叠后）。

### 4.2 进阶路径

| 目标 | 路径 |
|------|------|
| 提高一致性 | 设置·上下文开开关 → 保存 → 上下文库预建 → 批量翻译 |
| 术语 | 关键词 / 术语 → 提取 → 合并 glossary |
| 润色 | 订正 → 预览 → 写回订正 |
| 包过大 | 批量·执行·拆分 → 提交剩余包 |
| 检查需处理 | 批量·结果·问题处理 |
| 质量实验 | 诊断与工具 · A/B |
| 手写 CLI | 诊断与工具 · 命令参考 |

## 5. 分阶段交付

每阶段 **可独立合并**、有测试、更新用户可见文档。依赖：

```text
P0a（日志语义） → P0b（全局栏 + 写回折叠）
       ↓
      P1（左导航 + 任务页 + 页状态保留）
       ↓
      P2（能力归位 + 设置单源迁移）
       ↓
      P3（打磨）
```

**P0 硬拆**：P0a 与 P0b **不得**合并为「一个含糊的 P0 PR」，除非审阅明确同意且 diff 仍可分 commit 回滚；默认 **两个独立 PR**。

### P0a — 日志抽屉 + 焦点语义拆分（优先开工）

**问题背景（代码事实）**

今日 `MainWindow._focus_log_tab()`（`app.py`）**同时**做两件事：

1. `tab_widget.setCurrentWidget(diagnostics_tab)` — 强制切到诊断  
2. 动画放大 `diagnostics_splitter` 日志区 — 便于看原始输出  

调用面很广（非穷尽）：start / resume 工作流、bootstrap、template、写回 apply、recheck、repair、probe、split、A/B、部分 error 路径等。  
计划若只写「取消强制 focus，改为展开抽屉」却仍统一改 `_focus_log_tab`，会把 **已在诊断页启动的工具**（probe / split / A/B）所需的「放大诊断日志」一并打掉。

**范围**

1. **语义拆成两个 API**（名称可微调，职责不可混）：

   | API | 职责 | 典型调用方 |
   |-----|------|------------|
   | `_show_workbench_log_drawer()` | **不**切换主 Tab；展开工作台底部日志抽屉；确保 CLI 输出可见 | 工作台主路径：开始/继续翻译、写回、修补、bootstrap、template… |
   | `_expand_diagnostics_log()` | 若当前不在诊断 Tab，**可**切到诊断（或仅当已在诊断时放大）；动画放大诊断 splitter 日志区 | 从**诊断页工具条**启动的 probe / A/B / split；用户主动「在诊断中打开」 |

2. **废弃/降级 `_focus_log_tab`**：实现改为显式委托上述之一，或删除后全量替换调用点；禁止继续「一函数两语义」。

3. **工作台底部可折叠日志抽屉**：与现有 `log_view` **同源缓冲**（共享 document 或等价）；清空两边同步。

4. **调用点分类迁移**（验收用矩阵，实现时对照 `rg _focus_log_tab`）：

   | 场景 | 目标行为 |
   |------|----------|
   | 批量/同步翻译 start、resume | 留在工作台 → `_show_workbench_log_drawer()` |
   | 写回 apply、重新检查 | 同上 |
   | 同步修补 repair | 同上（结果摘要仍可写 diagnostics context，但不强切 Tab） |
   | 关键词/订正/预建等从工作台启动 | 同上 |
   | 诊断页 **试跑 probe** | 可留在诊断 → `_expand_diagnostics_log()` |
   | 诊断页 **拆分 split** | `_expand_diagnostics_log()` |
   | 诊断页 **A/B 对比** | `_expand_diagnostics_log()` |
   | 用户点抽屉「在诊断中打开」 | 切诊断 + 可选 `_expand_diagnostics_log()` |
   | runner error | 优先抽屉可见；若当前已在诊断则放大日志 |

5. 诊断页本身布局与工具条按钮集合：**本阶段不迁**拆分/试跑入口。

**非范围**：全局项目栏、写回折叠、左导航、glossary 入口调整、设置字段搬家。

**验收**

- [ ] 从工作台 **开始批量翻译**：主 Tab 仍为工作台；抽屉展开且有 CLI 输出；**不**自动切诊断

#### 页状态：相对今日 `_set_work_mode` 的硬约束

今日 `_set_work_mode()` 会：

- `self._workflow = None`
- 清空 `_workflow_step_output_lines`
- `self._writeback_manifest_path = ""`
- `_clear_completed_manifest_snapshot()`
- 再 `_apply_work_mode_ui(...)`

在「单壳变形」模型下这可接受；在「多任务页」模型下，**切换导航再回来**若仍走同一清空逻辑，用户会丢失：

- 当前页 workflow / 进度摘要  
- 写回用 manifest 路径与 `can_apply`  
- 关键词 glossary candidates 路径与合并按钮可用态  
- 订正预览写回态  
- 「查看上次已完成任务」快照  

**P1 必须改为按页（或按 WorkMode）保留会话状态**，例如：

```text
PageSession / ModeSession:
  workflow, writeback_manifest_path, completed_snapshot,
  keyword_merge_candidates_path, last summaries, stage index, ...
```

切换导航 = **激活另一页的 session 并刷新该页 UI**，而不是全局 `None` 清空。  
仅在「显式新建任务」或「切换 game_root」时重置相关 session。


- 迁移完成标准：`rg "_focus_log_tab"` 无业务调用（或仅薄包装且单测锁定委托目标）

### 6.4 任务页会话状态（P1）

**不要**把导航切换实现成今日的全局 `_set_work_mode` 清空。

| 事件 | 期望 |
|------|------|
| 左导航 A → B | 冻结 A 的 session；激活 B 的 session 并渲染 |
| 左导航 B → A | 恢复 A 的 workflow / manifest / 写回态 / merge candidates / 阶段索引等 |
| 开始新任务（用户点开始） | 重置**当前页** session 的运行态，再启动 |
| `game_root` 变更 | 重置**所有页** session 或标 stale 并强制刷新 |
| 任务 running | 禁用导航切换（与今日禁用 combo 一致） |

最低保留字段（可按页裁剪）：`work_mode`、`workflow`、`writeback_manifest_path`、writeback/doctor/workflow 摘要、`keyword_merge_candidates_path`、`completed_manifest_snapshot`、批量 `stage`、split 选中包等。

### 6.5 设置字段单源（P2）

- 从高级「移出」时：更新 `grouped_advanced_fields` / category，或上下文页专用列表 + advanced 排除集（`_SETTINGS_CONTEXT_OWNED_KEYS` 一类）

### 6.6 运行中切换任务

- `_task_running` 时禁用左导航（P1）  
- 全局项目切换禁用（P0b §3.1.1）

### 6.7 主题与 objectName

## 11. 成功标准（Epic 完成时）

1. 新用户可在**不打开诊断**的情况下完成批量翻译主路径。  
2. 任意用户目标（批量 / 同步 / 术语 / 订正 / 上下文库）在左侧导航有**稳定、不变名**的入口。  
3. 写回翻译与写回订正、glossary 合并在 UI 上**语义分离**。  
4. 拆分、试跑、A/B 等高级能力**可发现但不挡路**。  
5. GUI 自动化测试绿；`docs/gui_workbench.md` 与实现一致。  
6. CLI 行为与安全闸门**零回归**。
