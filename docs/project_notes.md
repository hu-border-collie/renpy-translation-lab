# 项目说明

文档地图：[docs/README.md](README.md)

本文档记录仓库状态、边界、安全注意事项和适用人群。日常使用入口见根目录 `README.md`。

## 环境要求

- Python 3.11+
- `google-genai`，供同步脚本和 Batch 脚本使用
- 有效的 Gemini API Key
- Ren'Py 项目中的 `game/tl/schinese` 翻译目录

## 当前边界

目前项目更偏“高级用户工具 / 核心引擎”，暂未重点覆盖：

- 完整普通用户发行体验。可选 GUI 已有工作台（批量翻译、同步翻译、关键词、订正等模式），批量翻译主路径已完成真实项目烟测；但仍缺少打包安装器、多项目管理和更完整端到端 QA，不是零配置成品（见 `gui_qt/`、`docs/gui_workbench.md`）
- Excel / HTML 协作流
- 面向普通用户的零配置体验
- 完整的游戏解包 / 打包一体化发布流程
- 面向超大项目的完整 RAG 生产工作流，例如严格的波次式回灌编排、多阶段调度策略
- 完整的结构化剧情图谱生产工作流，例如自动 seed 生成、Neo4j 可视化导出

## 项目状态

这是一个仍在持续探索和改进中的个人项目。核心 Batch 翻译链路已经在约 11 万英文词规模的真实 Ren'Py 项目上完整跑通，可以按“高级用户稳定版”的口径用于有备份的真实项目试跑。

- 日常实验和更新会先在作者本地工作区中进行；这个公开仓库只同步已经跑通、适合公开发布的版本。
- 它不是已经打磨完成、可直接开箱使用的零配置产品。
- 不保证在所有环境下稳定运行。
- 更适合作为 Ren'Py 翻译工作台、核心引擎和进一步改造的基础。
- 项目开发过程中使用了 AI 辅助生成代码，整体方向、功能取舍、测试验证与集成决策由作者负责。
- 目前不承诺及时处理 issue、兼容性问题或长期更新。
- 当前更推荐使用 Batch 脚本；同步脚本保留用于直接运行、补译、局部修复、smoke test，以及可选的 RAG 滚动记忆验证。
- 可选 PySide6 GUI 可以降低常规批量翻译流程的操作成本，但仍复用同一套 CLI、JSON 配置和 `check -> apply` 安全合约。2026-06-19，GUI **批量翻译**主路径已在一部约 3,300 待译行的真实 Ren'Py 项目上完成真实项目烟测：原文索引预建、批量提交/继续、下载、检查、可写回状态下写回和启动验证均跑通；首轮检查为「需处理」时 GUI 正确禁用写回，问题经 CLI / 人工修正后重新检查为「可写回」再写入项目（对应 CLI 的 `warn` → `safe`）。GUI 上的同步翻译、关键词提取、订正模式（#85 / #83 / #84）已实现并有自动化测试，但尚无各自独立的真实项目烟测记录。
- Batch / RAG 仍是主要验证方向；同步 RAG 更适合小批量即时反馈和局部精修，不是 Batch 吞吐流程的替代品。
- 当前的 RAG 能力更适合“小包验证 + 逐步扩展”，还不应被表述为已经完成的大项目生产级方案。
- Structured Story Memory 目前是可选 MVP，适合作为人工维护剧情上下文的 prompt 增强，不是完整自动图谱系统。

## 安全说明

执行任何会修改项目文件的操作前，请先备份，并优先在副本上测试。推荐先跑 `doctor`，再走 `build -> submit -> status -> download -> check -> apply`；只有 `check` 输出 `safe` 时才继续 `apply`。如果出现 `warn` / `block`，先查看失败报告并通过 retry / repair / revision 等流程处理。

不要把以下内容提交到公开仓库：

- 真实 API key
- 你本地的 `api_keys.json`
- 你本地的 `translator_config.json`
- 你本地的 `glossary.json` / `glossary_*.json`
- 你本地的 `story_graph.json` / `story_graph.seed.json`
- 你本地的 `macro_setting.md`
- 私有游戏脚本
- 本地 batch 结果
- history / rag store
- 日志和缓存

## 开发约定

本仓库的正式贡献规则见根目录 [CONTRIBUTING.md](../CONTRIBUTING.md)。

其中最重要的一条：**CLI 与 GUI 必须同步**。新功能、新配置、新诊断项、新安全闸门，应同时在命令行和图形工作台可用；核心逻辑只实现一次，GUI 通过共用模块或同一 CLI 参数接入。仅有意的例外（纯主题、调试命令等）才允许只改一侧，且须在 PR 中说明。

## 适用人群

这个仓库更适合下面这类使用者：

- 已经熟悉 Ren'Py 项目目录结构
- 能自行准备 `work/game/tl/schinese`、安装 Ren'Py SDK，或理解 `prepare` 行为
- 能阅读 Python 脚本并按需修改本地配置
- 接受这是高级用户工具，而不是稳定打包好的最终产品
