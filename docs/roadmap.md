# Ren'Py + Gemini 翻译项目开发路线图

## 1. 项目定位

这个项目目前最适合定义为：

**Ren'Py 翻译核心引擎 + Gemini Batch 作业系统 + 轻量 RAG 记忆层**

它和常见的 Ren'Py 翻译工具相比，重点不在 GUI、RPA 解包打包、Excel 协作这些外围能力，而在下面三件事：

1. **翻译请求更聪明**
   通过前文 / 目标区 / 后文、锁定术语、历史记忆等方式，让模型更容易翻对、翻顺、翻出角色感。

2. **Batch 生命周期更完整**
   不只是“导出 JSONL 去跑”，而是已经具备 `build / submit / status / download / check / apply / split / repair` 这类完整流程。

3. **已有翻译能沉淀为记忆**
   通过本地 RAG / history store，把过去译文检索回灌到后续请求中，提升一致性。

## 2. 优先级最高的 5 个模块

1. 语法级提取与精确回写层
2. 翻译后质检 / 自动修复层
3. 角色设定 / 术语 / 文体记忆层 2.0
4. 轻量人工校对工作台
5. 配置 / 环境 / 回归测试整合层

## 3. 分阶段开发路线图

### 一周内能做的事
- 扩充 `check` 规则
- 增加字段 / 标签 / 百分号 / 换行一致性检查
- 给 `apply` 增加更明确的安全门槛
- 给 manifest 和中间产物补更清晰的日志
- 准备最小 golden test 数据集

### 一个月内值得做的事
- 重构任务抽取与定位方式
- 给每个翻译单元建立更稳定的唯一标识
- 实现更强的 repair 规则
- 把角色卡 / 文体策略接入 prompt 构造
- 做一个最小审校器（HTML 或本地网页均可）
- 让 RAG 检索更可控

### 更长期再做的事
- 做图形化配置向导
- 补多模型 / 多后端支持
- 增加 Excel / HTML / JSON 导入导出
- 做更完善的术语管理界面
- 加项目级统计与审校仪表盘

## 4. 总结

一句话总结：

**先把发动机做到很强，再决定要不要造整车。**
